## 1. 시나리오 제목
* 내 손으로 만드는 종합 영화관 예매 및 결제 관리 시스템

<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 영화관을 방문한 고객의 예매 정보를 입력받아 최종 결제 금액을 산출하고 영수증을 발급하는 '종합 영화관 예매 및 결제 관리 시스템'입니다.
* 사용자는 원하는 영화 코드, 좌석 위치, 인원수, 단가 및 할인율을 입력합니다. 시스템은 이를 바탕으로 총액과 할인이 적용된 최종 금액을 계산하여 상세한 영수증 형태로 출력합니다. 이후 단계에서는 조건문을 통해 연령 제한 및 조조/심야 등 복합적인 할인 조건을 판별하고, 반복문을 적용하여 '예매하기', '취소하기' 등이 가능한 무한 루프 메인 메뉴를 도입할 예정입니다. 최종적으로는 배열을 이용해 시각적인 영화관 좌석표(예: 5X5)를 구현하고 여러 명의 예매 내역을 동시에 관리하며, 함수로 각 기능을 모듈화하여 완성도 높은 콘솔 예매 프로그램을 완성할 계획입니다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 영화 및 좌석 정보 입력
2. 최종 결제 금액(할인율 적용) 산출
3. 연령에 따른 관람 가능 여부 및 등급 판별
4. 시각적 영화관 좌석표 출력 및 예매 현황 조회
5. 영화관 메인 메뉴 시스템(예매, 취소, 조회, 종료)

  * 사용자로부터 영화 코드, 좌석 위치(열, 행), 예매 인원수, 티켓 1장 단가, 할인율을 입력받아 총액과 최종 결제 금액을 계산하여 영수증 형태로 출력합니다.
  * 영화 코드와 좌석 열(문자형), 좌석 행과 인원수 및 단가(정수형), 할인율과 최종 결제 금액(실수형)으로 자료형을 나누어 설계 및 구현했습니다. 
  * 사칙연산을 활용하여 '총액' 및 '최종 금액'을 계산하고, 제어문자(\n, \t)를 활용해 깔끔한 영수증 포맷으로 화면에 출력했습니다.

## 🟩 [2차 과제: V2.0] 조건 분기 적용 (조건문) - 향후 작성 예정
### **✨2차 과제 업데이트 내용:**
  * 단일 관람객 기준이었던 1차 과제에서 발전하여, 다수 일행의 연령대(청소년, 일반, 경로)인원을 각각 분리하여 입력받고 별도의 할인율을 적용하는 현실적인 예매 시스템으로 업그레이드함.
  * switch-case와 if-else 조건문을 적극 활용하여 영화코드(A,B,C)및 시간대(조조,일반,심야)에 따른 요금 분기 및 중복 할인 처리 로직을 구현함.
  * 할인율 한도 초과(100% 이상)를 방지하고, 잘못된 입력값(없는 영화 코드나 시간대 등)이 들어왔을 때 엉터리 영수증이 출력되는 것을 막기 위해 조건문 흐름 제어(if (ticket_price > 0)를 통한 예외 처리 및 방어적 프로그래밍 기법을 도입함.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:**  "2주차 진도(조건문) 제약에 맞춘 예외 처리 로직 설계를 바탕으로 복합 연령대(청소년/일반/경로) 중복 할인 적용 코드 구현 요청"
    * **적용 내용:** 잘못된 값을 입력했을 때 프로그램이 강제 종료되거나 0원 영수증이 출력되는 것을 막기 위해 if (ticket_price > 0) 조건으로 전체 실행 흐름을 안전하게 제어함. 또한 반복문을 사용할 수 없는 한계를 극복하기 위해, 청소년, 일반, 경로 인원을 각각 별도로 입력받고 개별 할인율(나이+시간대)을 계산한 뒤 할인 한도(최대 100%)를 방어하는 정교한 연산 로직을 완성함.
    
 2. **내용 2**
    * **프롬프트 요약:**  "코드 직접 타이핑 후 발생한 다수의 컴파일 에러 원인 분석 및 실수형 출력 포맷(%.0f%%)의 작동 원리 질문"
    * **적용 내용:** scanf, ticcket_price등의 오타와 세미콜론/부등호 오류 등을 디버깅하여 빌드 에러를 완벽히 해결함. switch-case 문에서 대소문자(A,a)를 동시에 처리하기 위해 폴스루(Fall-through)기법을 올바르게 적용함. 또한 %%이스케이프 시퀀스와 %.0f를 활용해 콘솔 영수증의 출력 포맷을 깔끔하게 다듬음.
    
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 잘못된 영화 코드나 시간대를 입력했을 때, 경고 메시지만 출력되고 계산 로직이 멈추지 않아 '최종 결제 금액:0원' 이라는 논리 오류(엉터리 영수증)가 발생하는 현상 발견.
     * **해결:** 아직 함수 종료(return 1)를 배우지 않았음을 감안하여, 초기화된 단가 변수를 활용해 if (ticket_price > 0) 구조로 묶어 정상직언 입력일 때만 게산 로직이 실행되도록 제어함. 조건문만으로 흐름을 완벽하게 통제하는 정석적인 예외 처리 방식을 체득함.
       
     **문제 2:** switch-case문에서 소문자와 대문자를 한 번에 처리하기 위해 case'A', 'a': 형태로 작성했으나 문법 오류가 발생함.
     * **해결:** C언어는 하나의 case에 콤마로 여러 값을 묶을 수 없다는 점을 확인하고, break;를 의도적으로 생략하여 아래로 조건이 이어지게 만드는 폴스루 (Fall-through)기법 (case 'A': csae 'a':)을 학습하여 가독성 좋고 에러 없는 코드로 수정함.

## 🟨 [3차 과제: V3.0] 무한 루프와 메뉴 시스템 (반복문) - 향후 작성 예정
### **✨3차 과제 업데이트 내용:**
  * 단발성으로 종료되던 기존 프로그램에 while(1) 무한 루프를 도입하여, 실제 영화관 키오스크처럼 예매가 끝나도 다음 고객을 위해 초기 화면으로 돌아가는 지속 가능한 메뉴 시스템을 구축.
  * 프로그램의 덩치가 커짐에 따라 가독성과 유지보수성을 높이기 위해 전체 로직을 4개의 함수(show_kiosk_screen, get_ticket_price, update_kiosk_system, make_reservation)로 분리 및 모듈화.
  * '함수 원형 선언(Prototype)'방식을 도입하여 main 함수를 파일 최상단에 배치함으로써 프로그램의 전체 뼈대를 한눈에 파악할 수 있도록 구조를 개선.
  * g_ticket_number(고유 발권 번호)와 g_remaining_seats(잔여 좌석 수)를 전역 변수로 선언하고, 예매가 완료될 때마다 이를 갱신하여 여러 번의 예매 사이클에도 데이터가 현실적으로 누적되고 유지되도록 상태 관리 기능을 구현.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:**  "main 코드를 위로 올리고 함수의 정의를 밑으로 둔 뒤, 정상 실행을 위해 함수 선언을 main 코드 위에 적는 구조의 적합성 질문 및 코드 적용 요청
    * **적용 내용:** c언어 실무에서 권장하는 '함수 원형 선언'방식을 학습하고 도입. 컴파일러가 함수를 인식하지 못해 발생하는 에러를 방지함과 동시에, 프로그램의 핵십 흐름인 main함수를 위로 끌어올려 코드의 가독성과 구조적 완성도를 크게 높임.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 과제 필수 평가 요소인 '반복문 탈출(break)'을 구현하려면 메인 메뉴에 '프로그램 종료' 버튼을 만들어야 했으나, 실제 고객용 키오스크 화면에 시스템 종료 버튼이 대놓고 노출되는 것은 매우 비현실적이라는 설계상의 딜레마가 발생
     * **해결:** 메인 화면의 안내 문구를 "1번을 눌러 영화 예매를 진행하세요"로 직관적으로 변경하여 사용자를 유도하고, 과제 제출용 종료 기능은 안내판에는 적혀있으나 일반 고객은 누를일이 없는 '0번 (관리자용 셧다운)' 메뉴로 분리하여 구현함. 이를 통해 과제 채점 기준(break 작동)을 100% 충족하면서도 현실의 UI/UX 기획 의도를 지켜내는 영리한 해결책을 체득.
     **문제 2:** 잔여 좌석을 차감하는 로직을 처음 구현했을 때, 상영관에 남은 좌석보다 더 많은 인원의 예매를 요청하느느 경우(예: 남은 좌석은 2석인데 일행 4명 예매 시도) 잔여 좌석 수가 음수로 떨어지며 그대로 결제까지 진행되는 치명적인 데이터 무결성 오류가 발생.
     * **해결:** make_reservation 함수 내에서 총 관람 인원(people_count)을 산출한 직후, 전역 변수인 g_remaining_seats와 비교되는 예외 처리 조건문 (if (people_count > g_remaining_seats) )을 추가. 좌석이 부족할 경우 경고 메시지와 함께 return;을 실행하여 함수를 조기에 종료시킴으로써 엉터리 발권을 원천 차단하고 시스템의 방어적 프로그래밍 수준을 한 단계 끌어올림.
